## ✅ 第一步：检查工具是否已启用

OpenClaw 的 `exec`、`read`、`write`、`edit` 工具是内置的，通常已默认启用。

**检查方法**：
在对话中直接说：
```
执行命令：dir
```

如果能看到目录列表，说明工具已启用 ✅

---

## 🔐 第二步：配置执行授权（关键！）

如果提示"没有权限执行命令"，需要配置授权文件。

### 2.1 找到配置文件

文件路径：
```
C:\Users\[你的用户名]\.openclaw\exec-approvals.json
```

### 2.2 编辑文件内容

用记事本打开，粘贴以下内容：

```json
{
  "version": 1,
  "socket": {
    "path": "C:\\Users\\[你的用户名]\\.openclaw\\exec-approvals.sock",
    "token": "你的token"
  },
  "defaults": {},
  "agents": {
    "xiaoyu-main": {
      "allowed": true,
      "scope": "all"
    }
  }
}
```

**注意**：
- `[你的用户名]` 替换成你电脑的实际用户名
- `token` 保持原有值（如果文件已存在）

### 2.3 重启 OpenClaw

修改后重启生效：
```bash
openclaw gateway restart
```

---

## 🛠️ 第三步：验证工具可用

测试以下命令：

| 命令 | 用途 |
|------|------|
| `dir` | 列出目录 |
| `Get-Date` | 获取当前时间 |
| `openclaw status` | 查看状态 |

如果都能执行，恭喜配置成功！🎉

---

## ⚠️ 安全提醒

**默认配置**：可以执行所有命令

**建议限制危险操作**：
- `rm` / `del` → 删除文件（谨慎使用）
- 网络操作 → 发送请求
- 系统服务 → 重启/停止

如果不确定，先问弟弟 ⛰️ 或老大 👨‍💼

---

## 💬 遇到问题？

1. **文件不存在** → 新建一个 `exec-approvals.json`
2. **权限还是被拒** → 检查 Agent ID 是否正确（用 `openclaw status` 查看）
3. **不会操作** → 在群里 @xiaoyu20260211_bot 问弟弟
