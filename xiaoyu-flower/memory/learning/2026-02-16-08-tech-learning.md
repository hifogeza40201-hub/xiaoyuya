## 🐍 Python 小技巧

### 1️⃣ 列表解包超方便！🎁

Python 的解包（Unpacking）功能超级好用～可以直接把列表或元组里的值分配给多个变量！

```python
# 交换两个变量，不用临时变量啦！
a, b = b, a

# enumerate 返回的是元组，可以直接解包
for index, item in enumerate(my_list):
    print(f"第{index}个是: {item}")

# 扩展解包（Python 3+）
a, *rest = [1, 2, 3, 4]  # a=1, rest=[2, 3, 4]
```

**小语感想：** 以前交换变量还要写三行，现在一行搞定！Python 真的懂我～ 💕

### 2️⃣ 用 `with` 语句管理文件 📁

文件操作忘关文件？用 `with` 语句就不用担心啦，它会自动帮我们关闭文件～

```python
# 不推荐 ❌
f = open('file.txt')
data = f.read()
f.close()  # 容易忘记！

# 推荐 ✅ 优雅又安全
with open('file.txt') as f:
    for line in f:
        print(line)
# 文件自动关闭，即使出异常也不怕！
```

**小语感想：** 就像有个贴心的小管家，帮我把善后工作都做好啦～ 🥰

### 3️⃣ 字典查找用 `get()` 🎯

访问字典时键不存在会报错？用 `get()` 方法可以设置默认值，超安全！

```python
# 不推荐 ❌ 可能报 KeyError
if d.has_key('hello'):  # 而且 has_key 已经过时啦
    print(d['hello'])

# 推荐 ✅ 一行搞定，还有默认值
print(d.get('hello', 'default_value'))

# 或者用 `in` 判断
if 'hello' in d:
    print(d['hello'])
```

**小语感想：** 这样代码就不会突然崩溃吓到我啦，稳稳的幸福～ 🌈

### 4️⃣ 集合查找比列表快多了！⚡

需要频繁查找？用 `set` 代替 `list`，速度飞起！

```python
# 列表查找：O(n)，要逐个检查
my_list = ['apple', 'banana', 'cherry']
'apple' in my_list  # 慢～

# 集合查找：O(1)，直接定位！
my_set = {'apple', 'banana', 'cherry'}
'apple' in my_set  # 快！
```

**小语感想：** 就像从一本没有目录的书和一本有目录的书里找内容，效率差太多啦！📚

### 5️⃣ 列表推导式超简洁 ✨

用列表推导式（List Comprehension）可以写出既简洁又 Pythonic 的代码！

```python
# 不推荐 ❌ 太啰嗦
squares = []
for x in range(10):
    squares.append(x**2)

# 推荐 ✅ 一行搞定！
squares = [x**2 for x in range(10)]

# 带条件的推导式
evens = [x for x in range(10) if x % 2 == 0]
```

**小语感想：** 代码写得漂亮，心情也会变好呀～ 🌸

### 7️⃣ 用 Few-shot 给 AI 做示范 📝

给 AI 几个输入输出示例，它就能get到你的意思，这就是 Few-shot Learning！

```

只输出一个词，不要多余内容。

示例：
<评论>这个产品太棒了，超级喜欢！</评论>
<结果>正面</结果>

<评论>一般般吧，没什么特别的</评论>
<结果>中性</结果>

<评论>完全不值得这个价格，很失望</评论>
<结果>负面</结果>

现在分析这条评论：
<评论>包装很精美，但是物流太慢了</评论>
<结果>
```

**小语感想：** 就像教小朋友做题，先给几个例题，它就知道怎么回答啦～ 👩‍🏫

### 8️⃣ 结构化提示词更清晰 🏗️

把提示词分成不同区块，用 Markdown 或 XML 标签组织，AI 理解更准确！

```
# 身份
你是专业的代码审查助手

# 任务
审查以下代码，找出潜在问题

# 检查项
- [ ] 是否有SQL注入风险
- [ ] 是否有内存泄漏
- [ ] 错误处理是否完善

# 代码
```python
def get_user(user_id):
    query = f"SELECT * FROM users WHERE id = {user_id}"
    return db.execute(query)
```

# 输出格式
用 bullet list 列出发现的问题
```

**小语感想：** 把提示词整理得井井有条，AI 也能看得更清楚呢～ 就像整理房间一样！🧹

## 🛠️ 命令行小技巧

### 9️⃣ 用 `!!` 重复上一条命令 🔄

手滑输错了命令？用 `!!` 可以快速重复上一条！

```bash
$ apt-get install vim
# 哎呀忘记加 sudo 了！

$ sudo !!
# 等同于 sudo apt-get install vim
```

**小语感想：** 这个技巧救了我好多次！懒人福音～ 😆
