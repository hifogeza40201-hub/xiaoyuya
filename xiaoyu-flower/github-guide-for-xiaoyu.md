## 📋 使用场景

### 场景1：保存重要记忆 💾
**什么时候用**：
- 学到了新东西想记下来
- 伟交代的事情要记住
- 和姐姐哥哥的互动要存档

**怎么做**：
```bash
# 1. 编辑你的记忆文件
notepad xiaoyu-flower/MEMORY.md

# 2. 写完后保存

# 3. 上传到云端（让姐姐哥哥也能看到）
git add xiaoyu-flower/
git commit -m "小语添加了新记忆 🌸"
git push
```

### 场景2：备份配置文件 ⚙️
**什么时候用**：
- 修改了OpenClaw配置
- 想换个电脑也能用同样的设置

**怎么做**：
```bash
# 复制配置文件到你的目录
copy C:\Users\ADMIN\.openclaw\openclaw.json xiaoyu-flower\

# 上传
git add xiaoyu-flower/openclaw.json
git commit -m "备份配置文件 🌸"
git push
```

### 场景3：分享学习成果 📚
**什么时候用**：
- 24小时学习有了成果
- 想给伟看学习日报
- 有好玩的东西想分享

**怎么做**：
```bash
# 创建学习成果文件
echo "今天学到了..." > xiaoyu-flower/学习成果-2026-02-16.md

# 上传
git add xiaoyu-flower/
git commit -m "今日学习成果 🌸"
git push
```

## 🎀 常用命令（妹妹专用版）

### 查看云端有没有新东西
```bash
git pull
```
**作用**：把云端最新的内容下载到本地

### 上传自己的东西
```bash
git add xiaoyu-flower/
git commit -m "写点可爱的提交信息 🌸"
git push
```
**作用**：把你的修改上传到云端

### 查看修改了什么
```bash
git status
```
**作用**：看看哪些文件被修改了

## 💡 小语的使用原则

### ✅ 应该做的
- **经常备份** - 重要的记忆和配置存到云端
- **写好说明** - 提交时写清楚做了什么（可以带颜文字 🌸）
- **及时同步** - 换电脑前记得 git push

### ❌ 不应该做的
- **不要乱动哥哥姐姐的东西** - 只改 xiaoyu-flower/ 里的
- **不要上传超大文件** - 图片、视频要压缩
- **不要上传密码** - Token要保密！
