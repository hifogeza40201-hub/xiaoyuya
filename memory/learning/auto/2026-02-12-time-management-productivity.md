## 📚 核心方法论

### 一、GTD (Getting Things Done) —— 搞定一切

**创始人**: David Allen  
**核心理念**: 把大脑从"记住事情"中解放出来，专注于"执行"

#### 五步法

| 步骤 | 英文 | 中文 | 核心动作 |
|------|------|------|----------|
| 1 | Capture | 收集 | 把所有想法、任务从大脑中"倒出"到可靠系统 |
| 2 | Clarify | 澄清 | 处理每个项目：是什么？下一步行动？ |
| 3 | Organize | 组织 | 分类到清单：项目、下一步行动、等待中、 someday |
| 4 | Reflect | 回顾 | 定期（每天/每周）回顾，保持系统更新 |
| 5 | Engage | 执行 | 基于情境、时间、精力、优先级选择行动 |

#### 两分钟法则
> 如果一件事能在2分钟内完成，**立刻做**，不要放进待办清单。

**对小雨的启发**: 帮助伟建立任务收集系统，不让他大脑过载。

---

### 二、艾森豪威尔矩阵 —— 区分紧急与重要

**来源**: 美国总统艾森豪威尔  
**核心**: 紧急 ≠ 重要

```
                    紧急                    不紧急
              ┌──────────────────┬──────────────────┐
              │                  │                  │
    重要      │   第一象限        │   第二象限        │
              │   【立即做】      │   【计划做】      │
              │   危机、deadline  │   学习、规划     │
              │   客户紧急需求   │   关系维护       │
              │                  │                  │
              ├──────────────────┼──────────────────┤
              │                  │                  │
    不重要    │   第三象限        │   第四象限        │
              │   【委托他人】    │   【删除不做】    │
              │   某些会议       │   刷社交媒体     │
              │   不重要的邮件   │   无意义忙碌     │
              │                  │                  │
              └──────────────────┴──────────────────┘
```

#### 关键洞察
- **第一象限**（紧急重要）：如果这里的事太多，说明你缺乏规划
- **第二象限**（重要不紧急）：这是**高价值区域**，投资时间在这里
- **第三象限**（紧急不重要）：学会委托，不要自己做
- **第四象限**（不紧急不重要）：果断删除

**对伟的启示**: 伟的工作涉及客户紧急需求（第一象限），但也要留出时间给产品规划（第二象限）。

### 三、帕累托法则（80/20法则）

**创始人**: 经济学家 Vilfredo Pareto  
**核心**: 80% 的结果来自 20% 的努力

#### 实际案例
- 80% 的销售来自 20% 的客户
- 80% 的 bug 来自 20% 的代码
- 80% 的财富集中在 20% 的人手中

#### 如何应用

**步骤1: 识别你的20%**
- 列出你做的所有事情
- 找出哪20%产生了80%的价值

**步骤2: 时间块保护20%**
- 在日历上为这20%的任务预留时间
- 不被打断，专注执行

**步骤3: 减少或委托80%**
- 那些低价值的80%，考虑：
  - 不做
  - 简化做
  - 委托他人做

**对伟的启示**: 伟的20%可能是——与核心客户建立深度关系、开发能解决真实痛点的AI产品。要保护这些时间。

### 四、番茄工作法 —— 专注的魔法

**创始人**: Francesco Cirillo（1980年代末）  
**名字来源**: 他用的番茄形状计时器

#### 工作流程

```
┌─────────────────────────────────────────────────────┐
│  选择一个任务 → 设置25分钟计时 → 专注工作           │
│         ↓                                           │
│  计时结束 → 休息5分钟 → 完成4轮后休息20-30分钟     │
└─────────────────────────────────────────────────────┘
```

#### 为什么有效
1. **时间压力**: 25分钟创造了一个"紧迫感"
2. **专注边界**: 告诉自己"只专注25分钟"，更容易开始
3. **休息恢复**: 定期休息防止 burnout
4. **量化工作**: 大任务可以估算"需要几个番茄"

#### 适用场景
- ✅ 需要持续专注的工作（写作、编程、学习）
- ✅ 容易拖延的大任务
- ✅ 想培养专注习惯的人

#### 不适用场景
- ❌ 需要长时间深度沉浸的创造性工作（可能打破心流）
- ❌ 高度不可预测的紧急任务

**对小雨的启示**: 可以用番茄工作法帮伟拆分大任务，比如"开发第一个AI功能"拆成多个25分钟的探索。

### 五、时间块（Time Blocking）

**核心理念**: 把你的日历变成你的待办清单

#### 如何操作
1. **每天早上/前一天晚上**，在日历上为任务分配具体时间块
2. **保护这些时间块**，像保护重要会议一样
3. **预留缓冲时间**，不要排满100%

#### 进阶技巧

**主题日**: 给每一天设定一个主题
- 周一：客户沟通日
- 周二：产品开发日
- 周三：行政处理日
- ...

**批处理**: 把同类任务集中处理
- 批量回复邮件（而不是随时回复）
- 批量开会（把会议集中在某一时段）

**对伟的启示**: 伟可以设置"客户支持时段"和"深度工作时段"，避免被频繁打断。

### 六、深度工作（Deep Work）

**提出者**: Cal Newport（《深度工作》作者）  
**定义**: 在无干扰状态下进行的专业活动，将认知能力推向极限

#### 深度工作 vs 浅度工作

| 深度工作 | 浅度工作 |
|---------|---------|
| 需要高度认知 | 低认知负荷 |
| 产生高价值 | 容易被复制/外包 |
| 心流状态 | 多任务切换 |
| 难以量化时间 | 容易填满时间 |

#### 深度工作原则

1. **拥抱无聊**: 不要一有空就刷手机，训练大脑适应无聊
2. ** quit social media**: 或至少限制使用
3. **把深度工作变成仪式**: 固定时间、固定地点、固定流程
4. **像经营企业一样经营时间**: 有投入产出比的思维

#### 四种深度工作哲学

| 哲学 | 描述 | 适合人群 |
|------|------|----------|
**隐居式** | 长时间（数周）完全脱离浅层工作 | 作家、研究者 |
| **双峰式** | 一年中部分时间深度工作，部分时间处理浅层工作 | 教授、顾问 |
| **节奏式** | 每天固定时间深度工作（如早上4小时） | 大多数上班族 |
| **记者式** | 随时随地抓住碎片时间进入深度状态 | 记者、高管 |

**对伟的启示**: 伟可以采用"节奏式"——每天固定时段（如上午）做深度工作（产品规划、AI功能开发），下午处理客户沟通。

## 💡 对伟的具体建议

### 短期（本周可开始）
1. **建立收集系统**: 用一个工具（Notion/Obsidian/滴答清单）收集所有想法和任务
2. **每天早上5分钟**: 用艾森豪威尔矩阵规划当天任务
3. **尝试番茄工作法**: 选一个拖延的任务，用25分钟开始

### 中期（1-3个月）
1. **识别你的20%**: 哪些客户、哪些工作产生了大部分价值？
2. **设立深度工作时间**: 每天固定2-4小时不受打扰的时段
3. **批处理低价值任务**: 把邮件、琐事集中在特定时段

### 长期（形成习惯）
1. **每周回顾**: 检视时间分配，调整策略
2. **持续优化**: 找到最适合自己的组合
3. **教会团队**: 如果伟有团队，分享这些方法
