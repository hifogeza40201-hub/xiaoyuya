## 🎯 核心内容概览

### 1. 边缘计算架构与优势

**三层架构模型:**
```
┌─────────────────────────────────────┐
│           云端层 (Cloud)            │  ← 全局分析、模型训练、长期存储
│    大数据平台 / AI训练 / 可视化     │
└─────────────────────────────────────┘
                  ↑
           安全回传通道
                  ↓
┌─────────────────────────────────────┐
│           边缘层 (Edge)             │  ← 本地推理、实时处理、数据过滤
│    边缘节点集群 / 规则引擎 / AI推理  │
└─────────────────────────────────────┘
                  ↑
           工业网络/5G/LoRa
                  ↓
┌─────────────────────────────────────┐
│          设备层 (Device)            │  ← 数据采集、执行控制
│    传感器 / PLC / 边缘网关 / 执行器  │
└─────────────────────────────────────┘
```

**核心优势:**
- ⚡ 低延迟: 毫秒级响应，满足实时控制需求
- 📉 省带宽: 本地预处理减少90%+数据传输
- 🔒 更安全: 敏感数据不出域
- 🌐 可离线: 断网自治能力

### 2. IoT协议对比

| 协议 | 传输层 | QoS | 适用场景 |
|------|--------|-----|---------|
| **MQTT** | TCP | 3级 | 不稳定网络、发布订阅 |
| **CoAP** | UDP | 2级 | 受限设备、RESTful |
| **HTTP** | TCP | - | Web集成、大数据量 |
| **WebSocket** | TCP | - | 实时双向通信 |
| **DDS** | UDP | 多级 | 实时系统、工业 |

### 3. 嵌入式系统基础

**软件栈层次:**
```
应用层 → 中间件(RTOS/协议栈) → HAL → 硬件
```

**主流平台:**
- **ESP32**: WiFi/BT IoT首选
- **STM32**: 工业控制标准
- **Raspberry Pi**: 边缘网关平台
- **Jetson**: AI边缘计算

### 4. 实际部署案例

| 案例 | 技术栈 | 效果 |
|------|--------|------|
| 智能制造预测维护 | 振动传感器+Jetson+LSTM | 故障预测>85% |
| 智慧农业监测 | LoRa+太阳能+土壤传感器 | 节水30% |
| 智慧城市路灯 | PLC+Mesh+光感控制 | 节能60-70% |
