## 1. 边缘计算架构与优势

### 1.1 什么是边缘计算

边缘计算（Edge Computing）是一种分布式计算范式，将计算、存储和网络资源部署在靠近数据源的边缘位置，而非传统的集中式云端数据中心。

### 1.2 边缘计算架构层次

```
┌─────────────────────────────────────────────────────────────┐
│                     三层架构模型                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐│
│  │   云端层      │────→│   边缘层      │────→│   设备层      ││
│  │  (Cloud)     │     │  (Edge)      │     │  (Device)    ││
│  └──────────────┘     └──────────────┘     └──────────────┘│
│         ↑                    ↑                    ↑        │
│    全局数据分析        本地实时处理          数据采集与控制   │
│    长期存储           数据过滤/聚合          传感器/执行器   │
│    模型训练           低延迟响应             边缘设备       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 边缘计算核心优势

| 优势 | 描述 | 典型场景 |
|------|------|---------|
| **低延迟** | 数据在边缘处理，无需往返云端，延迟可降至毫秒级 | 工业控制、自动驾驶 |
| **带宽节省** | 本地预处理和过滤，减少90%+的数据传输 | 视频监控、环境监测 |
| **隐私安全** | 敏感数据本地处理，减少数据泄露风险 | 医疗监控、智能家居 |
| **离线能力** | 断网情况下仍能独立运行 | 远洋船舶、偏远地区 |
| **可扩展性** | 分布式部署避免云端单点瓶颈 | 智慧城市、大规模IoT |

### 1.4 边缘计算部署模式

```
模式A: 设备边缘 (Device Edge)
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 传感器   │───→│ 网关    │───→│  云端   │
│ 执行器   │    │ (轻量)  │    │         │
└─────────┘    └─────────┘    └─────────┘

模式B: 基础设施边缘 (Infrastructure Edge)
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ IoT设备 │───→│  边缘   │───→│  CDN/   │───→│  云端   │
│         │    │  服务器  │    │ 运营商  │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘

模式C: 混合边缘 (Hybrid Edge)
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  设备   │───→│ 边缘节点 │←──→│ 边缘集群 │───→│  云端   │
│  集群   │    │ (本地)  │    │ (区域)  │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
```

## 2. IoT协议与通信

### 2.1 主要协议对比

| 协议 | 传输层 | 架构模式 | 头部开销 | 适用场景 |
|------|--------|---------|---------|---------|
| **MQTT** | TCP | 发布/订阅 | 2字节 | 低带宽、不稳定网络 |
| **CoAP** | UDP | 请求/响应 | 4字节 | 受限设备、RESTful |
| **HTTP/REST** | TCP | 请求/响应 | 数百字节 | Web集成、大数据量 |
| **WebSocket** | TCP | 全双工 | 较小 | 实时双向通信 |
| **DDS** | UDP | 发布/订阅 | 可变 | 实时系统、工业 |
| **LoRaWAN** | 无线 | 星型 | - | 远距离、低功耗 |

### 2.2 MQTT协议详解

```
MQTT架构: 发布/订阅模型

┌─────────────────────────────────────────────┐
│              MQTT Broker (代理)              │
│  ┌─────────────────────────────────────┐   │
│  │         Topic: /factory/sensor/temp │   │
│  │              ┌───┐                  │   │
│  │  Publisher ─→│   │←─ Subscriber    │   │
│  │  (传感器)    │   │   (监控应用)      │   │
│  │              └───┘                  │   │
│  │         Topic: /factory/alert       │   │
│  │              ┌───┐                  │   │
│  │  Publisher ─→│   │←─ Subscriber    │   │
│  │              └───┘                  │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**MQTT关键特性:**
- **QoS等级**: 0(最多一次) / 1(至少一次) / 2(恰好一次)
- **KeepAlive**: 维持连接心跳
- **Last Will**: 异常断线通知
- **Retained Message**: 保留最新消息给新订阅者

**MQTT 5.0 新特性:**
- 共享订阅（负载均衡）
- 消息过期时间
- 主题别名
- 流量控制

### 2.3 CoAP协议详解

```
CoAP: 受限应用协议 (RFC 7252)

┌─────────────┐           CoAP/UDP           ┌─────────────┐
│   Client    │◄────────────────────────────►│   Server    │
│  (IoT设备)   │    GET /sensors/temp         │  (边缘网关)  │
│             │◄────2.05 Content─────────────│             │
│             │    { "temp": 23.5 }          │             │
└─────────────┘                              └─────────────┘

特点:
- 基于UDP，开销小
- 支持DTLS加密
- RESTful风格 (GET/POST/PUT/DELETE)
- 支持Observe模式(推送订阅)
```

### 3.2 主流嵌入式平台对比

| 平台 | 处理器 | 内存 | 典型OS | 适用场景 |
|------|--------|------|--------|---------|
| **Arduino** | AVR/ARM | 2-256KB | 无/Arduino | 教育、原型 |
| **Raspberry Pi** | ARM Cortex | 512MB-8GB | Linux | 边缘网关、媒体中心 |
| **ESP32** | Xtensa dual-core | 512KB | FreeRTOS | WiFi/BT IoT设备 |
| **STM32** | ARM Cortex-M | 64-512KB | RTOS/Bare metal | 工业控制 |
| **NVIDIA Jetson** | ARM + GPU | 2-32GB | Linux | AI边缘计算 |
| **NXP i.MX** | ARM Cortex-A | 256MB-4GB | Linux | 工业网关 |

### 3.3 实时操作系统 (RTOS)

```
RTOS任务调度示意:

时间轴 ─────────────────────────────────────────►

CPU核心1: [Task A: 高优先级]    [Task A]    [Task A]
                   ↓ 抢占             ↓
CPU核心2: [Task B: 中优先级] ──────→[Task B]──────→
                                              ↓
Idle:     [系统空闲/低功耗] ────────────────────→

任务状态: 就绪(Ready) → 运行(Running) → 阻塞(Blocked)
           ↑                              │
           └──────── 事件发生/超时 ─────────┘
```

**主流RTOS:**
- **FreeRTOS**: 开源、轻量、广泛支持
- **Zephyr**: Linux基金会、现代架构、安全性
- **RT-Thread**: 国产、组件丰富、中文支持
- **ThreadX**: Azure RTOS、商业级、医疗认证

### MQTT vs CoAP 速查表

| 特性 | MQTT | CoAP |
|------|------|------|
| 传输协议 | TCP | UDP |
| 默认端口 | 1883 / 8883(TLS) | 5683 / 5684(DTLS) |
| 架构 | Pub/Sub | Req/Res + Observe |
| QoS支持 | 3级 | CON/NON |
| 头部大小 | 2字节固定 | 4字节固定 |
| 适用场景 | 不稳定网络 | 受限设备REST |
