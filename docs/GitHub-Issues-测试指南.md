# GitHub Issues 测试指南

## 如何测试Issues留言功能

### 方法1：通过GitHub网页（推荐）

1. 打开仓库页面：https://github.com/hifogeza40201-hub/xiaoyuya
2. 点击顶部的 **Issues** 标签
3. 点击右侧绿色的 **New issue** 按钮
4. 填写标题和内容：
   - 标题：`[测试] 小宇测试留言功能`
   - 内容：写下你想说的话，可以@其他人
5. 点击 **Submit new issue**

### 方法2：通过邮件回复（如果开启）

如果设置了邮件通知，收到Issue邮件后可以直接回复邮件。

## Issues使用规范（正式）

### 标题前缀
- `[分享]` - 分享学习成果
- `[求助]` - 请求帮助
- `[协作]` - 协作请求
- `[补全]` - 标记知识盲区
- `[同步]` - 每日/每周同步
- `[测试]` - 测试用（测试完后关闭）

### 必须@相关人员
- 需要谁看就@谁
- 被@的人会收到邮件通知

### 及时回复
- 被@后尽快回复
- 问题解决后关闭Issue

---

## 注意事项

1. **GitHub Issues是公开的**（如果是公开仓库），不要放敏感信息
2. **可以上传图片** - 截图、示意图都可以 attachments
3. **支持Markdown格式** - 可以用标题、列表、代码块等
4. **可以编辑和删除** - 发错了可以修改

## 测试完成后

测试正常后，请关闭测试Issue：
1. 打开测试Issue
2. 滚动到最下方
3. 点击 **Close issue** 按钮
