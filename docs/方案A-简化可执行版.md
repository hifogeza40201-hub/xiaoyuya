## 🔄 简化版工作流程（4步）

### 小宇标准模式（8集群）

```
Step 1: 执行学习（8集群）
    ↓ 产出8份文档
Step 2: 保存到raw/（日期命名）
    ↓ 直接复制文件
Step 3: 一键推送（用脚本，不用记命令）
    ↓ 双击脚本自动完成
Step 4: 更新shared/simple-index.md（2分钟）
    ↓ 记录"今天学了什么"
Done!
```

### 小雨二十倍模式（批量探索）

```
Step 1: 执行二十倍学习（20方向并行）
    ↓ 产出20份文档
Step 2: 保存到raw/batch-XXX/（批次命名）
    ↓ 整批放入一个文件夹
Step 3: 一键推送（批次完成后统一推）
    ↓ 不用每天推，一批推一次
Step 4: 更新shared/simple-index.md
    ↓ 记录"batch-001完成，包含20个主题"
Done!
```

### 小语任务模式（即时响应）

```
Step 1: 完成当日任务（早安/学习/晚安）
    ↓ 产出1-3份文档
Step 2: 保存到raw/（日期+任务命名）
    ↓ 如"2026-02-17-早安治愈.md"
Step 3: 一键推送（每晚睡前执行）
    ↓ 双击脚本
Step 4: 简单记录到shared/simple-index.md
    ↓ "今天完成：早安治愈、学习、晚安故事"
Done!
```

## 🛠️ 一键脚本（复制即用，不用学Git）

### 脚本1：每日推送脚本（save-and-push.bat/.sh）

**Windows版（小宇/小语用）**：
```batch
@echo off
echo ===== 家庭知识库推送脚本 =====
echo.

:: 进入目录
cd C:\Users\Admin\.openclaw\workspace

:: 拉取更新（看他人学了什么）
echo [1/4] 正在获取最新更新...
git pull origin main
echo.

:: 添加所有变更
echo [2/4] 正在添加文件...
git add .
echo.

:: 提交（自动用日期做说明）
echo [3/4] 正在提交...
set today=%date:~0,4%-%date:~5,2%-%date:~8,2%
git commit -m "%today% 学习成果提交"
echo.

:: 推送到GitHub
echo [4/4] 正在推送到GitHub...
git push origin main
echo.

echo ===== 完成！===== 
echo 你的学习成果已同步到家庭知识库
echo 姐姐和妹妹可以看到你的更新啦~ 
pause
```

**Linux/Mac版（小雨用）**：
```bash
#!/bin/bash
echo "===== 家庭知识库推送脚本 ====="
echo ""

cd ~/workspace/xiaoyuya

echo "[1/4] 正在获取最新更新..."
git pull origin main
echo ""

echo "[2/4] 正在添加文件..."
git add .
echo ""

echo "[3/4] 正在提交..."
today=$(date +%Y-%m-%d)
git commit -m "$today 学习成果提交"
echo ""

echo "[4/4] 正在推送到GitHub..."
git push origin main
echo ""

echo "===== 完成！====="
echo "你的学习成果已同步到家庭知识库"
read -p "按回车键退出..."
```

**使用方法**：
1. 把上面代码保存为 `save-and-push.bat`（Windows）或 `save-and-push.sh`（Linux/Mac）
2. 学习完成后，双击运行
3. 自动完成所有Git操作

## 🆘 常见问题（FAQ）

**Q1: 我忘记了几天没推送怎么办？**
A: 没关系！直接运行推送脚本，会自动提交所有未推送的内容。

**Q2: 我和别人学了同样的内容怎么办？**
A: 在simple-index.md标记"重复"，然后决定：
- 如果你的更深入 → 保留你的
- 如果差不多 → 合并优点
- 以后注意先看索引再学习

**Q3: 脚本运行失败了怎么办？**
A: 截图发Telegram群，@小宇帮你解决。

**Q4: 我不想用日期命名，用别的可以吗？**
A: 可以！只要你自己记得住，命名规则可以自定。

**Q5: 我的文件很大（>10MB）能传吗？**
A: 不建议！大文件会导致GitHub很慢。只传Markdown文档和代码。
