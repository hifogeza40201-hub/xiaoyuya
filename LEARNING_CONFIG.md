## 📊 质量评估标准

### 评估维度
| 指标 | 权重 | 测量方式 |
|------|------|---------|
| 文档大小 | 40% | 产出文件KB数 |
| 知识点数量 | 30% | 核心知识点计数 |

```

   ├── 文档 >50KB
   ├── 知识点 >15个

   └── 决策：下次可跳过，延长间隔

   ├── 文档 20-50KB
   ├── 知识点 8-15个

🔴 不足 (<70分)
   ├── 文档 <20KB
   ├── 知识点 <8个
   ├── 代码示例 <2个
   └── 决策：加强学习，缩短间隔
```

## ⚡ 即时触发机制（自主触发）

### 触发条件

#### 条件1：老大提问技术问题
```
检测关键词：
- "怎么实现"
- "如何配置"
- "报错"
- "优化"
- "设计"
- "架构"
- "性能"
- "安全"
- "部署"

动作：
→ 立即分析技术领域
→ 触发2-4个Agent集群学习
→ 15分钟内产出参考方案
```

#### 条件2：群聊技术讨论
```
检测场景：
- 讨论具体技术选型
- 分享技术文章/工具
- 遇到技术问题求助

动作：
→ 静默学习相关技术
→ 积累知识到MEMORY.md
→ 必要时主动提供建议
```

#### 条件3：执行任务失败
```
检测场景：
- 命令执行报错
- API调用失败
- 配置验证错误

动作：
→ 立即分析错误原因
→ 学习相关解决方案
→ 更新TOOLS.md避免重复错误
```

### 触发后流程
```
检测触发条件
    ↓
评估紧急程度（高/中/低）
    ↓
┌─────────┬─────────┬─────────┐
│  高     │   中    │   低    │
│ 2Agent  │  4Agent │ 4Agent  │
│ 立即    │  5分钟内│ 15分钟内│
│ 产出摘要│  完整报告│ 深度研究│
└─────────┴─────────┴─────────┘
```

## 🏷️ 特殊标记机制

### 标记类型
| 标记 | 含义 | 触发加强学习 |
|------|------|-------------|
| 🔥 重要 | 核心技术知识点 | 是，深度研究 |

| 🐛 故障 | 从错误中学习 | 是，诊断+修复 |
| 📌 项目 | 与当前项目相关 | 按需执行 |
| ⚡ 快速 | 只需概览了解 | 否，简化执行 |

### 标记方法
在对话中检测：
- "这个很重要" → 标记 🔥
- "新学的" / "没接触过" → 标记 🆕
- "又报错了" → 标记 🐛
- "项目要用" → 标记 📌
- "简单了解" → 标记 ⚡
